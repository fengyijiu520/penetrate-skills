# Java依赖漏洞审计
扫描Java项目的依赖，检测第三方组件的已知CVE漏洞，如Log4j、Fastjson、Fastjson等高危组件的漏洞。
---
## 核心检测规则
### 1. Maven依赖检测
检测pom.xml中的依赖版本，识别已知的CVE漏洞。
### 2. Gradle依赖检测
检测build.gradle中的依赖版本，识别已知的CVE漏洞。
### 3. 组件版本检测
识别组件的版本，匹配NVD的CVE数据库，检测高危漏洞。
### 4. 误报过滤
检测是否存在漏洞的补丁，过滤已修复的场景。
---
## 工作流程
1. 识别项目的依赖管理文件
2. 提取所有依赖的版本信息
3. 匹配NVD的CVE数据库，识别已知漏洞
4. 验证是否存在补丁，过滤误报
5. 生成审计报告
