# Java文件读取审计
扫描Java项目源码，检测任意文件读取、路径遍历漏洞，覆盖文件操作、压缩包处理等场景。

**无过滤审计**：本技能不会过滤任何文件，只要代码中存在漏洞，就会全部标记，不会排除任何文件，所有的风险都会被检测到。
---
## 核心检测规则
### 1. 基础文件操作检测
检测FileInputStream、FileReader等文件读取Sink的路径参数风险。
### 2. 路径拼接风险检测
检测basePath+userInput的路径拼接，无规范化验证的风险。
### 3. Zip Slip漏洞检测
检测压缩包解压时的路径遍历风险，如ZipEntry.getName()的不安全使用。
### 4. 工具类文件操作检测
检测FileUtils、IOUtils等工具类的文件读取风险。
### 5. NIO文件操作检测
检测Files.readAllBytes、Paths.get等NIO接口的路径风险。
### 6. 文件下载接口检测
检测文件下载接口的路径参数风险。
### 7. 反编译审计
支持对Jar包中的class文件进行反编译，审计字节码中的文件读取风险。
### 8. 误报过滤
通过getCanonicalPath验证、执行条件分析，过滤误报。
---
## 工作流程
1. 识别文件操作接口，定位用户输入源
2. 检测路径参数是否存在用户可控输入
3. 验证是否存在路径规范化、黑名单/白名单验证
4. 分析执行条件，过滤误报
5. 生成审计报告

## 输出
审计完成后，**默认直接生成美化的HTML报告**，无需额外指定，你不需要再说明"生成HTML"，系统会自动生成：
1.  **无过滤标记**：所有漏洞都已标记，没有排除任何文件
2.  **分类标注**：每个漏洞都标注了「文件读取」的分类标签
3.  **美化的HTML页面**：使用渐变背景、卡片设计、漏洞等级着色，支持响应式查看
