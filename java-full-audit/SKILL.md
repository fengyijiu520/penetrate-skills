# Java全项目代码审计
对Java项目进行全覆盖的代码审计，严格按照标准审计流程执行，**不跳过任何步骤**，依次完成所有专项审计，确保覆盖所有风险场景。

**无过滤审计**：本技能不会过滤任何文件，只要代码中存在漏洞，就会全部标记，不会排除任何文件，所有的风险都会被检测到。
---
## 标准审计流程（严格按顺序执行，禁止跳过）
### 步骤1：SQL注入审计
首先执行`java-sql-audit`技能，审计项目中所有SQL执行入口，检测SQL注入漏洞，覆盖JDBC、MyBatis、Hibernate等框架的所有风险场景。
### 步骤2：文件读取审计
然后执行`java-file-read-audit`技能，审计项目中所有文件读取接口，检测任意文件读取、路径遍历、Zip Slip漏洞。
### 步骤3：文件上传审计
接下来执行`java-file-upload-audit`技能，审计项目中所有文件上传接口，检测任意文件上传、文件类型绕过漏洞。
### 步骤4：认证授权审计
然后执行`java-auth-audit`技能，审计项目中所有敏感接口的权限控制，检测权限绕过、IDOR、认证缺失漏洞，覆盖Shiro、Spring Security等框架。
### 步骤5：命令执行审计
然后执行`java-command-exec-audit`技能，审计项目中所有命令执行入口，检测命令注入、远程代码执行漏洞。
### 步骤6：XXE注入审计
接下来执行`java-xxe-audit`技能，审计项目中所有XML解析接口，检测XML外部实体注入漏洞。
### 步骤7：SSRF审计
然后执行`java-ssrf-audit`技能，审计项目中所有HTTP请求接口，检测服务端请求伪造漏洞。
### 步骤8：反序列化审计
然后执行`java-deserialization-audit`技能，审计项目中所有反序列化接口，检测反序列化RCE漏洞。
### 步骤9：XSS审计
接下来执行`java-xss-audit`技能，审计项目中所有输出接口，检测跨站脚本漏洞。
### 步骤10：JNDI注入审计
然后执行`java-jndi-audit`技能，审计项目中所有JNDI查找接口，检测JNDI注入漏洞。
### 步骤11：Fastjson漏洞审计
然后执行`java-fastjson-audit`技能，审计项目中的Fastjson依赖和用法，检测Fastjson反序列化RCE漏洞。
### 步骤12：Log4j漏洞审计
接下来执行`java-log4j-audit`技能，审计项目中的Log4j依赖和用法，检测Log4Shell漏洞。
### 步骤13：硬编码凭据审计
然后执行`java-hardcode-audit`技能，审计项目源码，检测硬编码的密码、密钥、API Key等敏感凭据。
### 步骤14：不安全配置审计
然后执行`java-config-audit`技能，审计项目的配置文件，检测不安全的框架配置、安全配置缺失。
### 步骤15：依赖漏洞审计
最后执行`java-dependency-audit`技能，审计项目的第三方依赖，检测已知的CVE漏洞。
---
## 工作流程
1. 识别项目的技术栈，加载所有专项审计的规则
2. **严格按顺序执行**，先完成SQL注入审计，再进行文件读取审计，依次执行，**禁止跳过任何步骤**
3. 每个专项审计完成后，自动汇总结果，再进行下一个专项
4. 所有专项审计完成后，汇总所有漏洞，生成完整的全项目审计报告
5. 报告中会标注每个专项的审计结果，确保无遗漏
---
## 使用方式
当你需要对Java项目进行全覆盖审计时，只需要输入：
```
审计当前Java项目，全覆盖
```
系统会自动调用本技能，严格按照上述流程，依次执行所有专项审计，不会跳过任何步骤，完成全项目的安全审计。

## 输出
审计完成后，**默认直接生成美化的HTML报告**，无需额外指定，你不需要再说明"生成HTML"，系统会自动生成：
1.  **无过滤标记**：所有漏洞都已标记，没有排除任何文件
2.  **分类标注**：每个漏洞都标注了所属的漏洞分类，方便你区分不同类型的风险
3.  **美化的HTML页面**：使用渐变背景、卡片设计、漏洞等级着色，支持响应式查看
4.  整合所有专项审计的结果，生成完整的报告
