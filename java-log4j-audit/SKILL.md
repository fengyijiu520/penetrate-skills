# Java Log4j审计
扫描Java项目中的Log4j依赖和用法，检测Log4j2的JNDI注入漏洞（Log4Shell）。
---
## 核心检测规则
### 1. 版本检测
检测Log4j的版本，识别是否存在Log4Shell的高危版本。
### 2. 日志参数检测
检测日志打印的参数是否存在用户可控输入，是否存在${jndi:}注入风险。
### 3. 配置检测
检测Log4j的配置，是否存在消息查找的配置风险。
### 4. 误报过滤
检测输入是否可信，过滤安全场景。
---
## 工作流程
1. 识别Log4j的依赖版本
2. 检测日志打印的参数，定位用户输入源
3. 验证是否存在注入风险
4. 分析配置，过滤误报
5. 生成审计报告
